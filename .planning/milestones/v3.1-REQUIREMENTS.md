# Requirements: v3.1-paper-strengthening

**Defined:** 2026-02-17
**Core Value:** Address 5 critical weaknesses in the paper to withstand peer review. Analysis must be rigorous enough for FAccT 2027.
**Strategy:** Code-first — add statistical rigor and model expansion to the pipeline, then rewrite paper sections to incorporate new evidence.

## Statistical Rigor (Weaknesses 3+4)

- [ ] **STAT-01**: Bootstrap confidence intervals for all subgroup FNR, FPR, precision, and PR-AUC estimates. Use survey-weighted bootstrap (resample PSUs, not individuals) with 1000+ replicates.
- [ ] **STAT-02**: Survey-weighted hypothesis tests for FNR/FPR disparity significance across language groups. Wald tests or Rao-Scott chi-squared with FACTOR07 weights. Report p-values for key comparisons (castellano vs each indigenous group).
- [ ] **STAT-03**: Confidence intervals for intersectional subgroup estimates. Flag groups where CI width exceeds 0.3 as "insufficient precision for inference."
- [ ] **STAT-04**: Update `fairness_metrics.json` with CI bounds (lower, upper) and p-values for each subgroup metric.

## Predictive Validity (Weakness 2)

- [ ] **VALID-01**: Lift analysis — compute lift of LightGBM over prevalence baseline at each decile. Show the model provides meaningful signal above base rate.
- [ ] **VALID-02**: Calibration-by-decile analysis — compute actual vs predicted dropout rate per probability decile, report calibration error.
- [ ] **VALID-03**: Brier skill score relative to prevalence-only baseline. Quantify predictive improvement.
- [ ] **VALID-04**: Discussion of what low PR-AUC means for fairness metric interpretability. Honest assessment with evidence from lift/calibration.

## Model Family Expansion (Weakness 5)

- [ ] **MODEL-01**: Train Random Forest with same features, temporal splits, and FACTOR07 weights. Tune via cross-validation or Optuna.
- [ ] **MODEL-02**: Train MLP (sklearn MLPClassifier) with same features, temporal splits, and FACTOR07 weights. Standardize features for NN.
- [ ] **MODEL-03**: Compute full fairness metrics (FNR, FPR, precision, PR-AUC per subgroup) for RF and MLP using fairlearn MetricFrame.
- [ ] **MODEL-04**: Update `model_results.json` with RF and MLP performance metrics.
- [ ] **MODEL-05**: Cross-model fairness comparison — table showing FNR by language group for all 5 models (LR, LightGBM, XGBoost, RF, MLP). Compute max FNR gap consistency across model families.

## Paper Rewrite (All Weaknesses)

- [ ] **PAPER-13**: Change title to accurately reflect proxy audit (e.g., "Equity Audit of Dropout Prediction in Peru: Who Gets Missed by Early Warning Models?")
- [ ] **PAPER-14**: Add ENAHO-vs-SIAGIE feature comparison table in Data section — show which features SIAGIE has (grades, attendance, disciplinary) vs what ENAHO provides (household/survey), discuss implications for external validity.
- [ ] **PAPER-15**: Reframe Introduction and Discussion — position paper explicitly as "what disparities CAN emerge" in dropout prediction, not "what Alerta Escuela DOES." Strengthen proxy caveat to be a central framing element, not a limitation footnote.
- [ ] **PAPER-16**: Report bootstrap CIs for all fairness metrics in tables and text. Flag n=89 urban-indigenous CI prominently. Rewrite intersectional narrative to lead with uncertainty.
- [ ] **PAPER-17**: Add Predictive Validity subsection in Results — lift analysis, calibration-by-decile, honest discussion of PR-AUC=0.236 vs baseline=0.134. Argue model adds signal (lift>1) even if absolute performance is modest.
- [ ] **PAPER-18**: Rewrite algorithm-independence section with RF+MLP evidence. If all 5 models show similar FNR patterns → genuine algorithm independence. If RF/MLP diverge → discuss what that means.
- [ ] **PAPER-19**: Add statistical significance results to fairness tables (asterisks or separate column). Discuss which disparities survive hypothesis testing and which don't.
