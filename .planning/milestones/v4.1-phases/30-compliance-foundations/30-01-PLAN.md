  <action>
Create `src/fairness/threshold_sweep.py` that:

1. Reuses `_load_and_prepare_data("lgbm_calibrated")` from `src/fairness/metrics.py` to get y_true, y_prob (calibrated), weights, merged_df for v1 model.
2. Also loads v2 data via `_load_and_prepare_data` with the v2 model name (check `_PRED_FILE_MAP` for the key -- likely need to add a v2 entry or load v2 predictions directly from `data/processed/predictions_lgbm_v2.parquet`).
3. Builds a harmonized language group array from merged_df using the same `HARMONIZED_LANGUAGE_MAP` pattern as `metrics.py` (lang_castellano, lang_quechua, lang_aimara, lang_other_indigenous columns). Exclude foreign (n=43).
4. Sweeps thresholds from 0.05 to 0.95 in steps of 0.05 (19 thresholds). At each threshold:
   - Compute binary predictions: y_pred = (y_prob >= threshold).astype(int)
   - For each language group, compute weighted FNR = weighted sum of (y_true==1 & y_pred==0) / weighted sum of (y_true==1). Use FACTOR07 weights.
   - Record the FNR rank order (1=highest FNR, i.e., most missed).
5. Repeat for v2 model.
6. Export to `data/exports/threshold_sweep.json` with structure:
   ```json
   {
     "metadata": {"thresholds": [...], "models": ["v1_25f", "v2_31f"]},
     "v1_25f": {"0.05": {"castellano": {"fnr": 0.xx, "rank": 1, "n_dropouts": N}, ...}, ...},
     "v2_31f": {"0.05": {"castellano": {"fnr": 0.xx, "rank": 1, "n_dropouts": N}, ...}, ...},
     "rank_invariant": true/false,
     "invariant_range": "0.05-0.95 (all thresholds)" or description of where it breaks
   }
   ```
7. Generate `paper/tables/table_13_threshold_sweep.tex` -- a LaTeX table showing thresholds (rows, sample every 0.10 for readability: 0.10, 0.20, ..., 0.90) x language groups (columns: Castellano, Quechua, Otros Ind., Aimara). Show FNR values with rank superscripts. Include a row for the optimal threshold. Add footnote about Aimara small sample (n=76). Use booktabs formatting consistent with existing tables.

Important: For v2 data loading, if `_load_and_prepare_data` doesn't support v2 directly, load `predictions_lgbm_v2.parquet` manually and join with features the same way metrics.py does. The v2 threshold is in `model_results.json` at `lightgbm_v2.threshold_analysis.optimal_threshold` or similar -- check the JSON structure.

If rank order does NOT hold at some thresholds, report honestly: note the range where castellano > quechua > other_indigenous holds and where it breaks. This is science, not advocacy.
